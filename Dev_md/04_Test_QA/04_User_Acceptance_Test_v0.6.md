# 🧪 사용자 직접 테스트 시나리오 (UAT Scenario) - v0.6

## 1. 개요
본 테스트는 실제 사용자가 PyTarot 서비스를 이용하는 전체 여정을 검증합니다.

## 2. 테스트 시나리오 (UAT-01: 표준 타로 리딩)

### 단계 1: 리딩 시작 (Session Creation)
- **사용자 행동:** 질문 입력("올해의 전반적인 운세가 궁금해요"), 스프레드 선택("쓰리 카드").
- **시스템 검증:**
    - `POST /api/v1/readings` 호출 성공.
    - 서버에서 중복 없는 3장의 카드 선별(Pre-draw).
    - 스프레드 설정에 맞는 위치(과거/현재/미래) 매핑 확인.

### 단계 2: 카드 드로우 및 선택 (Interaction Simulation)
- **사용자 행동:** 78장의 카드 덱(Fan-out) 중 3장 클릭.
- **시스템 검증:**
    - 프론트엔드에서 클릭 시 슬롯 이동 애니메이션 작동 확인.
    - 3장 선택 완료 시 결과 페이지 자동 전환 확인.

### 단계 3: 결과 표시 (Display Verification)
- **사용자 행동:** 결과 화면 진입.
- **시스템 검증:**
    - 카드 뒤집기 애니메이션 수행.
    - 카드 이름(한글), 이미지, 상하 방향(정/역) 표시 확인.
    - 각 위치별 의미(과거/현재/미래)가 카드와 매칭되어 출력됨.

### 단계 4: AI 심층 해석 (RAG-based Interpretation)
- **사용자 행동:** AI 해석 대기.
- **시스템 검증:**
    - `POST /api/v1/interpretations/stream` 호출.
    - **RAG 작동:** 지식 베이스(`data/knowledge/`)에서 뽑은 카드 3장의 정보를 정확히 검색.
    - **스트리밍:** AI 응답이 실시간(SSE)으로 끊김 없이 출력됨.
    - **내용 전문성:** 뽑힌 카드의 상징적 의미와 사용자의 질문이 논리적으로 연결된 전문적 조언 제공.
